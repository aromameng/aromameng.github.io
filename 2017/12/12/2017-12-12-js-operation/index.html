js运算整理

发表于 2017-12-12 | 分类于 javascript

整理下js中简单的四则运算，以后别又被搞晕了

加法操作

加法操作分为以下几种情况：

字符串+数字

数字转换成字符串，然后相加

字符串+ 布尔

字符串 + null 或undefined

对象 + 字符串或数字

对象会进行toString()转换

数字 + 布尔

数字 和 字符串 | 布尔 | 对象等 进行 -、*、% 操作

字符串会先转换成数字，然后再进行其他运算

其他减、乘、除等算法，都是先把字符串、布尔、对象转换成数字，然后再进行运算。如果转换失败，则会返回NaN

Number 和 parseInt的差异

Number （类型转换） 对不能转换成数字的字符串，返回NaN

parseInt （解析） 会忽略非数字字符，只返回数字

总结

  • “+” 也可以当做字符串连接符，因此，进行”+” 运算时，如果一边有字符串或者对象，则另外的类型会先转换成字符，再进行字符串连接操作。

    如果两边都无字符串或对象，则会当成是加法运算

  • 其他 “-“、 “*”、 “%” 等操作，都是非数字类型转为数字类型，再进行运算，转换失败则返回NaN
  • null 跟数字运算，会先转换成 0
  • undefined 跟数字运算，会返回NaN
