js类型转换

发表于 2017-12-12 | 分类于 javascript

整理下js中 ‘==’ 的类型转换，这里确实好绕。 ‘===’ 这个就不用介绍了，不会发生类型转换，直接进行比较

主要分两种情况，类型相同和类型不同进行比较：

x，y类型相同

这一块比较简单，数字比大小，字符串比大小。

需要注意的是 NaN != NaN ，另外对象比较又是个复杂的情况了

x，y类型不同

这里又分为很多种情况了：

x，y一方为布尔值

如果x，y其中一个是布尔值，那么对这个布尔值进行toNumber操作。

x，y为数字和字符串

将字符串的一方进行toNumber()操作

x，y 为 null 和 undefined

返回 true

x,y一方为对象

这种情况就比较复杂了，需要对Object进行 ToPrimitive()操作。

ToPrimitive 具体转换规则见 9.1小节： http://yanhaijing.com/es5/#103

里面写到对Object 进行 ToPrimitive ：“返回该对象的默认值。对象的默认值由把期望类型传入作为hint参数调用对象的内部方法[[DefaultValue]]得到”，

这个 [[DefaultValue]] 的解释见8.12.8小节：http://yanhaijing.com/es5/#100
